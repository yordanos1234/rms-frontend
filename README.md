## 📖 About the Project

**Registral (RMS)** is a full-stack **Registrar Management System** developed as a **Final Year Defense Project** at:

> **Addis Ababa University**  
> College of Natural & Computational Sciences  
> Department of Computer Science

This repository contains the **frontend client** — a responsive, role-based web application that digitizes registrar workflows for private campuses and polytechnic colleges. Students, instructors, registrar staff, department heads, and administrators each get a tailored dashboard and tools.

### ✨ What This Frontend Delivers

| Module | Description |
|--------|-------------|
| 🏠 **Landing Page** | Animated hero, project info, and call-to-action |
| 🔐 **Authentication** | JWT-based login with demo account cards |
| 📊 **Dashboard** | Role-specific stats, charts, and quick actions |
| 👨‍🎓 **Students** | Student records and enrollment management |
| 📚 **Courses** | Interactive course catalog with CRUD for authorized roles |
| 📝 **Grades** | Submission, approval workflow, and grade scale display |
| 📄 **Documents** | Transcript, enrollment letter, certificate & clearance requests |
| 📢 **Announcements** | Institutional news with audience targeting |
| 📈 **Reports** | Analytics charts + downloadable PDF reports |
| 👤 **Profile** | User profile and account details |

## 🚀 Quick Start

```bash
# Clone the repository
git clone https://github.com/yordanos1234/rms-frontend.git
cd rms-frontend

# Install dependencies
npm install

# Start development server
npm run dev
```

App runs at **`http://localhost:3000`**. API requests to `/api` are proxied to `http://localhost:5000` — start the [backend](https://github.com/yordanos1234/rms-backend) first.

### 📜 Scripts

| Command | Description |
|---------|-------------|
| `npm run dev` | Start Vite dev server |
| `npm run build` | Production build → `dist/` |
| `npm run preview` | Preview production build locally |

## 🏗️ Project Structure

```
frontend/
├── src/
│   ├── components/     # Reusable UI components
│   ├── context/        # Auth & global state
│   ├── pages/          # Route-level views
│   ├── App.jsx         # Router & layout
│   └── main.jsx        # Entry point
├── index.html
├── vite.config.js      # Dev proxy & build config
└── vercel.json         # SPA rewrite rules
```

## ☁️ Deployment

Build and deploy the `dist/` folder to **Vercel**, **Netlify**, or any static host:

```bash
npm run build
```

Configure your host to proxy `/api` requests to the deployed backend, or update the API base URL in your deployment config.
